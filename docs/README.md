## Documentation Standards

All documentation follows the **core-python-standards**:

- **English Only**: All documentation is in English
- **Detailed Descriptions**: Every component includes:
  - **Purpose (Why)**: Business need and purpose
  - **Implementation (What)**: Technical implementation details
  - **Parameters/Returns**: Complete parameter and return descriptions
  - **Raises**: List of potential exceptions
- **Examples**: Code examples for common use cases
- **Type Hints**: Full type information in code examples
